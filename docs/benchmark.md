# Benchmark Report

Generated by `go run ./cmd/benchmark`. Re-run the command to refresh this file after code changes.

The runner executes each transport/scenario sample in a fresh `go test` process so repeated runs stay stable on loopback. The heavy matrix lives in `cmd/benchmark`, while `internal/server` keeps only a small default benchmark set unless `EXPOSE_TRANSPORT_MATRIX_CASES` is provided. Each scenario measures `10`, `25`, `50`, or `100` requests per tunnel and uses about one requester worker per two tunnels, capped at `100`.

## How To Read It

| Metric         | Better  | Meaning                                                        |
| -------------- | ------- | -------------------------------------------------------------- |
| Sweep ms       | Smaller | Time to finish one full sweep of all requests in the scenario. |
| Req/s          | Larger  | Completed public HTTP requests per second for the full sweep.  |
| us/request     | Smaller | Average wall-clock cost per request within the sweep.          |
| KiB/request    | Smaller | Heap bytes allocated per request.                              |
| allocs/request | Smaller | Heap allocations per request.                                  |
